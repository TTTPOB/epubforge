# 标点规则

## 目标

这份规则文档描述的是“何时应该把标点视为同一风格问题”，而不是一套旧式人工脚本流程。

标点修复的首要目标有两个：

1. 不改变原文语义
2. 让同一部书在所有文本承载字段里保持一致

这里的“文本承载字段”不仅包括正文段落，也包括标题、脚注、表格标题、表格说明和表格 HTML 本体。

## 总原则

### 1. 先判定风格，再做替换

不要因为某个字符“看起来怪”就直接改。先回答两个问题：

- 这是不是全书已有的稳定约定？
- 这是不是同一语义功能下的异形字符混用？

如果答案仍不明确，scanner 应记录 convention 或 open question，而不是盲改。

### 2. 全书一致性高于局部顺手

单章里最常见的形式，不一定是全书主样式。特别是 range connector、破折号和中西文复合连接符，必须先看全书 inventory，再决定是否把局部章回归到主样式。

### 3. 语义类型不同，不能混修

至少要区分四类对象：

- 数字或日期范围连接符
- 叙述性破折号
- 术语或词内连接符
- 机器标记或协议字面量

它们可以长得像，但不应互相替换。

### 4. 任何脚注 marker 都是禁改区

像 `\x02fn-12-①\x03` 这样的 marker 不是普通文本。scanner 和 fixer 不应把其中的 `-`、数字或符号纳入标点统一。

## 规则

### 数字与日期范围

数字范围、日期范围、年龄区间、页码区间、表号等，应该使用同一套 range connector。做法不是硬编码某一种字符，而是：

1. 先识别本书主样式
2. 再把明显偏离的少数写法拉回主样式

常见风险：

- 表题和表体用不同连接符
- 同一章内 `1978-1980` 与 `1978—1980` 混用
- `表7-6` 与 `表7—6` 混用

只要它们表达的是“范围”或“编号连接”，就应保持同类一致。

### 叙述性破折号

叙述性破折号应按书内既有风格处理，不应被误判为 range connector。判断时看它承担的是语气停顿、插入说明还是数值范围。

如果一个字符两边不是数值或编号，而是语句成分，优先按破折号看待。

### 术语内部连接

英文复合词、音译词、术语内部连接符，通常不应因为全书的 range style 而被统一替换。它们属于词法层，不属于排版层。

如果中西文混排术语本身存在多种写法，需要 reviewer 做风格仲裁，再沉淀为 convention。

### 同一结构单元一起修

表格是最容易出现“只修一半”的地方。若某张表的范围符号需要统一，应同时看：

- `table_title`
- `caption`
- `html`

不要只修单元格而漏掉表题，也不要只修表题而放过表内主数据列。

## 常见模式

### 模式 A: range connector 混用

表现：

- 同一章或同一表中混用 `-`、`–`、`—` 等字符
- doctor 报告提示章节与全书主样式偏离

处理：

- 先把它记录为风格问题，而不是立即视为 OCR 错字
- 若证据明确，fixer 做成批、同语义、同字段族的修复

### 模式 B: 章节偏离全书主样式

表现：

- 单章 inventory 有明显 dominant dash
- 但与全书 dominant dash 不同

处理：

- 若该章是独立文体区块，例如附录、引文、外文材料，可保留为局部 convention
- 若只是抽取噪声，回归全书主样式

### 模式 C: 中西文连接符风格不稳定

表现：

- 同一术语族使用不同连接法
- scanner 无法确认这是作者风格还是抽取抖动

处理：

- 先生成 open question
- reviewer 裁决后再沉淀为 chapter-scoped 或 book-scoped convention

## 允许的修复

- 统一同一语义下的异形 dash
- 修复同一表内部明显不一致的范围连接符
- 修复标题、脚注、正文之间的同类风格漂移

## 不允许的修复

- 改写语气和句法，只因为“某种破折号更好看”
- 修改脚注 marker 或其他机器标记
- 在没有全书证据时，把局部特殊风格强行归一
- 把词内连接符误修成范围符，或反过来

## 何时升级为 reviewer

- 两种写法在全书中都大量存在，且无法判断主样式
- 某章明显是特殊文体，是否保留独立风格存在争议
- 某个连接符既可能是排版符号，也可能是作者有意保留的术语写法
