# 脚注规则

## 目标

脚注编辑的核心不是“把脚注文本修漂亮”，而是维护三种对象之间的关系：

1. 正文中的 callout
2. 机器 marker
3. 对应的 footnote body

只要这三者的关系错位，EPUB 的可点击跳转和语义归属都会出问题。

## 总原则

### 1. 一条脚注首先是一条链接关系

脚注不是孤立文本块。判断一条脚注是否正确，至少要同时看：

- footnote block 的 `callout`
- 宿主 block 中是否存在对应 marker
- 页码和上下文是否支持这条配对

### 2. 同页同符号是默认锚点

默认情况下，同页、同 callout 的正文宿主与脚注体应互相匹配。跨页配对可以存在，但必须有额外证据支持，而不是把跨页当常态。

### 3. raw callout residue 通常表示关系未闭合

如果正文或表格里还残留裸露的 `①`、`†` 等符号，而不是 marker，那么常见解释只有三种：

- 还没配对成功
- 配错后留下了残渣
- 原书本身就是印刷异常

在没有证据前，不要把它当作无害噪声。

### 4. `paired` 与 `orphan` 互斥

一条脚注不能既被标记为已配对，又被标记为 orphan。任何出现这类状态冲突的地方，都应视为结构错误。

## 规则

### marker 与 footnote body 一一对应

同一页、同一 callout 下：

- 不应出现多个正文 marker 争抢同一条脚注
- 不应出现多个脚注体共享同一个 callout

如果出现重复，先判断是抽取重复、错误 relink，还是原书排版异常。

### 跨页只在“关系更强”时成立

跨页脚注通常来自两类情况：

- 正文段落或表格本身跨页延续
- 书籍版面把脚注体挤到了下一页

这种情况下，应优先寻找最强证据的宿主，而不是简单按“最后一个见到的同符号”配对。

### 未抽到脚注体，不要伪造

如果正文里存在 callout，但没有任何合理的 footnote body，可接受的处理是：

- 保持为 unresolved 问题
- 在证据充分时标记 orphan
- 升级为 reviewer 判断是否属于原书错误

不可接受的处理是凭猜测编造脚注正文。

### 表格中的脚注与正文脚注同等重要

表格 cell、表题、表注里都可能承载 callout。不要把脚注宿主限制在普通段落上。

## 常见模式

### 模式 A: duplicate marker / duplicate body

表现：

- 同页同符号出现多个 marker
- 或多个 footnote body 共享同一 callout

这通常说明关系映射已经歧义化，需要 fixer 或 reviewer 明确唯一归属。

### 模式 B: marker without body

表现：

- 正文中已有 marker
- 但找不到同页同符号的脚注体

先检查是否跨页，再检查是否是抽取遗漏，最后才考虑原书缺失。

### 模式 C: raw callout residue

表现：

- 非脚注 block 内残留裸 callout 符号

这往往说明：

- 未配对
- 已配对但配错
- 曾经 unpair / relink 后留下未清理残渣

### 模式 D: paired without marker / orphan with marker

表现：

- 脚注体声称已配对，但宿主侧没有 marker
- 或脚注体标为 orphan，但宿主侧仍有 marker

这是典型的结构状态不一致，应优先修复。

### 模式 E: 异常高脚注密度

单页脚注密度异常高，不一定直接等于错误，但很适合作为 scanner 复查提示。密集页更容易发生重复符号、宿主误归属和跨页误配。

## 允许的修复

- 在证据明确时 `pair`、`unpair`、`relink`
- 在确认没有合理宿主时标记 orphan
- 消除因误配留下的 raw callout residue
- 修正 `paired` / `orphan` 状态冲突

## 不允许的修复

- 发明缺失的脚注正文
- 让一个脚注体同时服务多个宿主
- 仅凭最近邻原则处理跨页配对
- 为了消除告警而随意把 unresolved 问题标成 orphan

## 何时升级为 reviewer

- 同一 callout 有多个合理宿主，且语义上都说得通
- 书籍版面本身似乎存在印刷错误
- 跨页归属需要内容判断，而不是结构判断
- scanner 与 fixer 对 callout 集合或配对风格形成冲突 convention

## 与 memory 的关系

以下信息适合沉淀到 memory：

- 本书常用的脚注 callout 集
- 某类跨页脚注的稳定判定模式
- 已确认属于原书缺陷、无需重复修复的异常页
